# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

-リモートリポジトリ
  ・GitHubなどのサーバ上で管理可能なリポジトリ
  ・インターネット経由でアクセスする
  ・バックアップとして機能する
-ローカルリポジトリ
  ・自分のPC内にあるリポジトリ
  ・ローカルで動くためインターネット接続を必要としない
  ・作業中の変更はローカルリポジトリに保存される

## プッシュとマージの違いは何でしょうか？

-プッシュ
  ・ローカルリポジトリからリモートリポジトリへ情報の更新を行う際に使う
-マージ
  ・ブランチを切った先での作業をmasterに合流させる作業

## コミットとプッシュの違い

-コミット
  ・addによりステージングしたデータにコメントを添えるなどすることで、この時点での変更をしましたという履歴の作成を行う。
-プッシュ
  ・コミットしたデータをリモートリポジトリ上に送る操作を指す。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

-曖昧ではなく、どの時点で書いたか、どの理由で更新したかを添えること。どのような編集を行ったかを記入すること

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

-ローカル時
  ・自らのPC内でブランチ統合を行う。
  ・レビューを必要としない
  ・短時間でできるが品質は落ちる
-プルリクエスト時
  ・リモートリポジトリ上でブランチ統合をするように依頼
  ・マージ前にレビューが入ることがある。
  ・時間はかかるが、レビューなどを通すため品質は保証される。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

-先にマージされた変更内容を取り込む
-後にマージしようとしている変更内容を取り込む
-どちらの変更内容も取り込む
-上記のどれを適用したとしても処理上で困った際などはソースコードを書いた人とのコミュニケーションが必要となる。
